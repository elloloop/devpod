# Scaffold — Rust gRPC (Append-Only Module Architecture)

Set up a Rust gRPC service where adding a feature = adding a directory under `modules/`. Core is frozen. Modules self-register via the `inventory` crate.

## Architecture

```
src/
  main.rs                           ← FROZEN — starts server
  core/                             ← FROZEN after scaffold
    mod.rs
    server.rs                       ← tonic server, collects from inventory
    registry.rs                     ← Module trait + inventory registration
    bus.rs                          ← tokio broadcast event bus
    store.rs                        ← sqlx PgPool
    config.rs                       ← env-based config
  modules/                          ← APPEND-ONLY
    mod.rs                          ← ONE line append per module (pub mod invoice;)
    health/
      mod.rs                        ← inventory::submit! self-registration
    invoice/                        ← Agent A adds this
      mod.rs                        ← inventory::submit! to self-register
      handler.rs                    ← implements tonic service trait from gen/
      service.rs                    ← business logic
      store.rs                      ← sqlx queries
      events.rs
      tests.rs
gen/                                ← GENERATED by build.rs from proto — never hand-edit
proto/
build.rs                            ← FROZEN — compiles all proto/*.proto
Cargo.toml
```

### Self-registration via `inventory` crate

The `inventory` crate enables compile-time self-registration. Each module calls `inventory::submit!` — the core collects all registered modules without a hardcoded list.

### 2. Build script — `build.rs` (FROZEN)

```rust
fn main() -> Result<(), Box<dyn std::error::Error>> {
    // Compile all .proto files in proto/ directory
    let protos: Vec<_> = std::fs::read_dir("proto")?
        .filter_map(|e| e.ok())
        .filter(|e| e.path().extension().map_or(false, |ext| ext == "proto"))
        .map(|e| e.path())
        .collect();

    if !protos.is_empty() {
        tonic_build::configure()
            .build_server(true)
            .build_client(true)
            .out_dir("gen")
            .compile_protos(&protos, &["proto"])?;
    }

    Ok(())
}
```

### 3. Core — `src/core/registry.rs` (FROZEN)

```rust
use tonic::transport::Server;

/// Every module implements this trait and submits via inventory::submit!
pub trait Module: Send + Sync + 'static {
    fn name(&self) -> &'static str;

    /// Register gRPC services with the tonic server builder.
    /// Returns a modified Router with the service added.
    fn register_grpc(&self, router: tonic::transport::server::Router)
        -> tonic::transport::server::Router;

    /// Register event subscribers with the event bus.
    fn register_subscribers(&self, bus: &super::bus::EventBus);
}

// Collect all modules via inventory at compile time.
// No hardcoded list. Just implement Module + inventory::submit!
inventory::collect!(Box<dyn Module>);

pub fn all_modules() -> impl Iterator<Item = &'static Box<dyn Module>> {
    inventory::iter::<Box<dyn Module>>
}
```

### 4. Core — `src/core/server.rs` (FROZEN)

```rust
use tonic::transport::Server;
use tonic_health::server::health_reporter;
use super::registry::all_modules;
use super::bus::EventBus;
use super::config::Config;

pub async fn serve(cfg: Config) -> Result<(), Box<dyn std::error::Error>> {
    let addr = format!("0.0.0.0:{}", cfg.port).parse()?;
    let bus = EventBus::new();

    // Health service
    let (mut health_reporter, health_service) = health_reporter();

    // Start with a base server
    let mut router = Server::builder()
        .add_service(health_service);

    // Auto-register all modules from inventory
    for module in all_modules() {
        tracing::info!(module = module.name(), "registering module");
        router = module.register_grpc(router);
        module.register_subscribers(&bus);
        health_reporter.set_serving::<&str>(module.name()).await;
    }

    // Enable reflection
    let reflection = tonic_reflection::server::Builder::configure()
        .build_v1()?;
    router = router.add_service(reflection);

    tracing::info!(%addr, "gRPC server listening");
    router.serve(addr).await?;
    Ok(())
}
```

### 5. Core — `src/core/bus.rs` (FROZEN)

```rust
use std::collections::HashMap;
use std::sync::{Arc, RwLock};
use tokio::sync::broadcast;

#[derive(Clone, Debug)]
pub struct Event {
    pub event_type: String,
    pub payload: serde_json::Value,
}

type Handler = Arc<dyn Fn(Event) + Send + Sync>;

pub struct EventBus {
    handlers: RwLock<HashMap<String, Vec<Handler>>>,
}

impl EventBus {
    pub fn new() -> Self {
        Self { handlers: RwLock::new(HashMap::new()) }
    }

    pub fn subscribe<F>(&self, event_type: &str, handler: F)
    where
        F: Fn(Event) + Send + Sync + 'static,
    {
        let mut handlers = self.handlers.write().unwrap();
        handlers.entry(event_type.to_string())
            .or_default()
            .push(Arc::new(handler));
    }

    pub fn publish(&self, event: Event) {
        let handlers = self.handlers.read().unwrap();
        if let Some(handlers) = handlers.get(&event.event_type) {
            for handler in handlers {
                handler(event.clone());
            }
        }
    }
}
```

### 9. Health module — `src/modules/health/mod.rs`

```rust
use crate::core::registry::Module;
use crate::core::bus::EventBus;

struct HealthModule;

impl Module for HealthModule {
    fn name(&self) -> &'static str { "health" }

    fn register_grpc(&self, router: tonic::transport::server::Router)
        -> tonic::transport::server::Router
    {
        // Health is registered by core — this is a pattern example
        router
    }

    fn register_subscribers(&self, _bus: &EventBus) {}
}

inventory::submit! { Box::new(HealthModule) as Box<dyn Module> }
```

### 10. How agents add a module (the template)

`src/modules/invoice/mod.rs`:
```rust
mod handler;
mod service;

use crate::core::registry::Module;
use crate::core::bus::EventBus;
use crate::gen::invoice::invoice_service_server::InvoiceServiceServer;

struct InvoiceModule;

impl Module for InvoiceModule {
    fn name(&self) -> &'static str { "invoice" }

    fn register_grpc(&self, router: tonic::transport::server::Router)
        -> tonic::transport::server::Router
    {
        router.add_service(InvoiceServiceServer::new(handler::InvoiceHandler::default()))
    }

    fn register_subscribers(&self, bus: &EventBus) {
        // bus.subscribe("payment.completed", |e| { ... });
    }
}

inventory::submit! { Box::new(InvoiceModule) as Box<dyn Module> }
```

Then add `pub mod invoice;` to `src/modules/mod.rs`.
