Why devpod exists

We studied how the best teams ship code.
Then we built the tool.

The practices that make Meta, Google, and Stripe engineering teams fast aren't proprietary. They just weren't available outside those companies. Until now.

At Meta, engineers use stacked diffs via Phabricator. Instead of one massive PR, every change is a small, reviewable piece that ships independently. Reviews take minutes, not days.

Stacked Diffs — inspired by Meta's Phabricator workflow

At Google, every change gets a thorough code review with true interdiff between review rounds. Reviewers see exactly what changed since their last review, not the entire PR from scratch.

Version History — inspired by Google's Critique review tool

At Stripe, engineers run CI locally before pushing. Bugs are caught in seconds on their machine, not minutes later in a remote pipeline. The feedback loop is near instant.

Local CI — inspired by Stripe's engineering practices

These practices aren't secrets. They're just trapped inside custom internal tools that require massive infrastructure. devpod brings all of them to every team, as a single CLI.

FAQ for skeptics

Fair questions, honest answers.

We built devpod for ourselves first. These are the questions we asked too.

"Do I need to learn a new workflow?"
Four commands: feature, diff, sync, land. You'll be productive in 5 minutes. Everything else is optional.
"Does my team need to adopt this?"
No. devpod creates standard GitHub PRs. Your teammates can review on GitHub as usual. Only you need devpod installed.
"What about my existing .github/workflows?"
They run locally, unchanged. Same YAML, same marketplace actions. When they pass locally, they'll pass on GitHub.
"Is this just a git wrapper?"
Under the hood, yes. But so is gh. The value isn't the wrapper — it's the workflow: stacked diffs, version history, local CI, and a diff viewer that doesn't suck.
"What if I don't want stacked diffs?"
Don't use them. devpod feature + devpod diff + devpod land works as a single-diff flow too. Stack when you want, don't when you don't.
"Why not just use Graphite / spr / ghstack?"
They solve stacking on GitHub. devpod solves the entire workflow: CI, review, documentation, and yes — stacking. One tool, not five.
